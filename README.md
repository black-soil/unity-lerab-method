本项目是基于论文《行为即意图:智能体多维状态融合的可读性增强机制》中提出的 LERAB (Legibility Enhancement and Recognition of Agent Behaviors)​ 方法的 Unity 实现。我们重点复现了论文中的 强混淆场景 (Highly Confusing Scenario)，并集成了消融实验 (Ablation Study)​ 功能，用于验证多维状态融合机制 (MFRM) 在提升智能体行为可读性方面的有效性。

核心思想：传统方法主要通过让智能体产生偏离最优路径的“差异化行为”来传达意图，这在复杂或目标密集（强混淆）的场景下效率低下且可能失效。LERAB 方法的核心——多维融合表征机制 (MFRM)——不改变智能体原有的最优行为路径，而是通过融合智能体实时的姿态 (Posture)、速度 (Velocity)、距离 (Distance)​ 等多维度状态信息，来增强其行为对观察者（人或另一个智能体）的“可读性”，从而实现更快、更准确的意图识别，并提升任务执行效率。

✨ 核心特性

完整复现 LERAB 方法：实现了论文中提出的 MFRM 机制与基于贝叶斯更新的高效意图识别方案。

强混淆场景：精确还原了论文中设计的挑战性场景（目标点密集，路径高度重叠），用于测试方法在极端情况下的有效性。

消融实验选项：支持独立关闭“角度”、“距离”（颜色）、“速度”等状态维度的融合，以量化分析各维度对意图识别准确率的贡献。

可视化与数据分析：实时显示智能体路径、状态信息、观察者信念概率变化，并记录关键评估指标（平均正确识别率、最早意图传达步、任务成本）。

🗺️ 论文场景复现：强混淆场景 (图5)

本项目的实验环境严格遵循论文第4.1节“强混淆场景”的设置：

网格大小: 5 x 8

智能体起始点: 状态 16

可能目标点 (θ3): A(状态2), B(状态4), C(状态6), D(状态20)

智能体真实目标: C (状态6)

障碍物 (墙体): 位于状态 10, 12, 14

动作空间: 9个方向（上、下、左、右、左上、右上、左下、右下、停留）

此场景的挑战在于，前往不同目标（尤其是B, C, D）的路径在初期高度重叠，仅凭路径信息难以早期区分意图，从而完美验证MFRM机制的价值。

🚀 快速开始
系统要求

Unity Editor: 2022.3.46f1c1 (必须使用此版本以确保兼容性)

支持的平台：Windows, macOS, Linux (用于编辑器开发与运行)

安装与运行

克隆仓库

bash
git clone [https://github.com/black-soil/unity-lerab-method.git]
cd [https://github.com/black-soil/unity-lerab-method.git]

使用 Unity Hub 打开项目

在 Unity Hub 中，点击 Open-> Add project from disk，选择克隆下来的项目文件夹。

确保编辑器版本切换为 2022.3.46f1c1。

打开主实验场景

在 Unity 编辑器的 Project窗口中，导航至 Scenes文件夹。

双击打开 HighlyConfusingScene.unity。

运行与配置

点击编辑器上方的播放按钮 ▶️ 运行场景。

游戏运行时，可以在 Game视窗中观察智能体移动和可视化信息。

运行实验与查看结果

开始/停止：通过UI按钮或实验管理器组件中的按钮控制，空格键为启动智能体。

实时可视化：

Game视图：显示网格地图、智能体、目标、墙体及智能体移动路径。

信念概率图：在调试栏查看，实时显示观察者对智能体前往各目标（A, B, C, D）的信念概率。

状态信息：显示智能体当前的速度、与各目标的距离和角度。

结果输出：实验结束后，关键指标会输出：

Unity Console​ 窗口。
